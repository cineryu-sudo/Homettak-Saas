## 1. 프로젝트 개요

### 1.1 배경
홈앤뚝딱은 주방 설비(싱크볼, 수전, 음식물처리기 등) 판매 및 설치 서비스를 제공하는 사업입니다. 현재 고객 상담부터 설치 완료까지의 프로세스가 수동으로 운영되어 월 200건 이상의 주문을 처리하는 데 병목이 발생하고 있습니다.

### 1.2 목표
**고객 상담 → 정보 수집 → 기사 배정 → 정보 전달 → 입금 확인**의 전 과정을 자동화하여 운영 효율성을 극대화합니다.

### 1.3 핵심 가치
- **시간 절감**: 수동 데이터 입력 및 전달 시간 90% 감소
- **오류 감소**: 휴먼 에러로 인한 정보 누락/오기입 방지
- **확장성**: 주문량 증가에도 동일한 리소스로 대응 가능
- **고객 경험**: 신속하고 일관된 서비스 제공

## 2. 현황 분석 (As-Is)

### 2.1 현재 프로세스

```
[고객 문의] → [상담사 수동 응대] → [엑셀에 수동 입력] → [기사 수동 배정] 
    → [카톡/문자로 수동 전달] → [입금 수동 확인] → [설치 완료]
```

### 2.2 현재 데이터 구조 (엑셀 기반)

**월별 주문 시트 (12월, 11월 등)**
| 필드명 | 설명 | 자동화 대상 |
|--------|------|-------------|
| 대화 | 상담 채널 (네톡, 카톡 등) | 자동 기록 |
| 발주일 | 주문 날짜 | 자동 기록 |
| 고객명/아이디 | 고객 식별 정보 | 고객 입력 |
| 연락처 | 고객 전화번호 | 고객 입력 |
| 주소 | 설치 주소 | 고객 입력 |
| 구매상품/수량 | 주문 상품 | 고객 선택 |
| 주문현황 | 배송/설치 상태 | 자동 업데이트 |
| 시공 예정일 | 설치 희망일 | 고객 입력 |
| 설치담당자 | 배정된 기사 | 자동 배정 |
| 시공비용 | 설치 비용 | 자동 계산 |
| 추가작업 | 특이사항 | 고객/상담사 입력 |
| 결제금액 | 총 결제액 | 자동 계산 |
| 잔금 입금일 | 입금 확인일 | 자동 확인 |

**설치 기사 리스트 시트**
| 필드명 | 설명 |
|--------|------|
| 지역 | 담당 권역 (서울, 인천, 경기 등) |
| 활동 지역 | 상세 활동 구역 |
| 업체명 | 기사/업체명 |
| 연락처 | 기사 연락처 |
| 설치비용 | 제품별 설치 단가 |
| 계좌번호 | 정산 계좌 |

### 2.3 Pain Points
1. **수동 데이터 입력**: 카톡 상담 내용을 엑셀에 일일이 옮김
2. **기사 배정 지연**: 지역별 기사 확인 및 연락에 시간 소요
3. **정보 전달 누락**: 복사-붙여넣기 과정에서 오류 발생
4. **입금 확인 지연**: 계좌 수시 확인 필요
5. **현황 파악 어려움**: 실시간 주문/설치 현황 확인 불가

## 4. 기능 요구사항

### 4.1 카카오톡 채널봇 (고객 인터페이스)

#### FR-001: 상담 시작 및 메뉴 선택
- 고객이 채널 추가 시 환영 메시지 자동 발송
- 메뉴: 상품 문의 / 설치 예약 / 배송 조회 / AS 문의

#### FR-002: 설치 예약 정보 수집
챗봇이 순차적으로 수집해야 할 정보:
```
1. 고객명 (필수)
2. 연락처 (필수, 형식 검증)
3. 설치 주소 (필수, 주소 검색 API 연동)
4. 구매 상품 선택 (필수, 버튼/캐러셀 UI)
5. 설치 희망일 (필수, 캘린더 UI)
6. 희망 시간대 (오전/오후/저녁)
7. 추가 요청사항 (선택)
```

#### FR-003: 입력 정보 확인 및 수정
- 입력 완료 후 전체 정보 요약 표시
- 수정 기능 제공 (항목별 수정 가능)
- 최종 확인 버튼으로 접수 완료

#### FR-004: 접수 완료 알림
- 접수 번호 발급 및 안내
- 예상 배정 시간 안내
- 결제 안내 (계좌 정보, 금액)

### 4.3 자동화 워크플로우

#### FR-008: 지역 기반 자동 기사 배정
```python
# 배정 로직 의사코드
def assign_technician(order):
    region = extract_region(order.address)  # 주소에서 지역 추출
    available_techs = query_technicians(
        region=region,
        date=order.preferred_date,
        status='active'
    )
    
    # 우선순위: 1)지역 정확도 2)평점 3)당일 배정 건수
    ranked_techs = rank_by_priority(available_techs)
    
    if ranked_techs:
        return ranked_techs[0]
    else:
        return notify_admin("배정 가능 기사 없음", order)
```

**지역 매핑 규칙:**
| 주소 키워드 | 지역코드 | 배정 기사 풀 |
|-------------|----------|--------------|
| 서울 강남/서초/송파 | SEOUL_GANGNAM | 집수리플러스, 이정호 |
| 서울 강북권 | SEOUL_GANGBUK | 하츠힘펠설치팀장 |
| 경기 남양주/구리 | GYEONGGI_EAST | 이정호 |
| 경기 안산/시흥 | GYEONGGI_ANSAN | 제이엠스톤 |
| 인천 전체 | INCHEON | 제이엠스톤, 하츠힘펠 |

#### FR-009: 기사 알림 발송
**발송 채널:** 카카오 알림톡 (Primary) + SMS (Fallback)

**알림톡 템플릿:**
```

```

#### FR-010: 입금 확인 자동화
**방법 1: 토스페이먼츠 가상계좌**
- 주문별 가상계좌 발급
- 입금 시 웹훅으로 자동 확인
- Notion 상태 자동 업데이트

**방법 2: 계좌 스크래핑 (대안)**
- 정기적 계좌 조회 (5분 간격)
- 입금자명 + 금액 매칭으로 주문 연결
- 매칭 실패 시 관리자 알림

```

         정보 입력 (이름, 연락처, 주소, 상품, 희망일)

[Step 2] 채널봇 → Make 웹훅
         JSON 페이로드 전송

[Step 3] Make → Notion
         주문 DB에 신규 레코드 생성
         상태: "접수완료"

[Step 4] Make → 지역 추출 모듈
         주소 파싱 → 지역코드 반환

[Step 5] Make → Notion 기사 DB 조회
         지역코드 + 활성상태로 필터링

[Step 6] Make → 배정 로직 실행
         우선순위 계산 → 최적 기사 선정

[Step 7] Make → Notion 업데이트
         주문 레코드에 기사 연결
         상태: "기사배정"

[Step 8] Make → 카카오 알림톡 API
         기사에게 고객 정보 전송
         (실패 시 SMS Fallback)

[Step 9] 고객 → 입금
         가상계좌 또는 일반 계좌

[Step 10] 토스 웹훅 → Make
          입금 정보 수신

[Step 11] Make → Notion 업데이트
          입금상태: True
          입금확인일: 현재일시
```

### 6.3 Agent 구성 (Claude Code Projects)

#### Agent 1: 주문 접수 Agent
```markdown
# 역할
카카오톡 채널봇으로 들어온 고객 문의를 처리하고 주문 정보를 수집합니다.

# 사용 Tools
- create_order: 주문 생성
- extract_region: 지역 추출

# 워크플로우
1. 고객 메시지 수신
2. 의도 파악 (상품문의/설치예약/배송조회/AS)
3. 설치예약인 경우 정보 순차 수집
4. 입력 완료 시 주문 생성
5. 접수 완료 메시지 발송
```

#### Agent 2: 기사 배정 Agent
```markdown
# 역할
신규 주문에 최적의 설치 기사를 자동 배정합니다.

# 사용 Tools
- query_technicians: 기사 조회
- assign_technician: 기사 배정
- send_technician_notification: 알림 발송

# 배정 로직
1. 주소에서 지역코드 추출
2. 해당 지역 담당 기사 목록 조회
3. 우선순위 점수 계산:
   - 지역 정확도 (구/동 일치) +3점
   - 평점 4.5 이상 +2점
   - 당일 배정 건수 3건 미만 +1점
4. 최고 점수 기사 배정
5. 카카오 알림톡 발송 (실패 시 SMS)
```

```markdown
# 역할

3. 매칭 성공 시 상태 업데이트
4. 매칭 실패 시 관리자 알림
```
