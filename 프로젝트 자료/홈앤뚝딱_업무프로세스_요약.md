# 홈앤뚝딱 업무 프로세스 요약 정리

> 녹음본 기반 2인 대화 정리 | 업무 자동화 설계용

---

## 1. 고객 유입 경로

### 1-1. 주요 유입: 네이버 검색 광고 (95% 이상)
- 고객이 "사각 싱크볼" 등 키워드를 네이버에서 검색 (예: 월평균 조회수 2,400건)
- 조회수 높은 키워드(예: "싱크볼 설치" 약 14,000건)를 선별하여 검색 광고 키워드로 세팅
- 광고 타이틀에 키워드를 삽입 → 검색 시 상위 노출
- 고객이 광고를 클릭 → 리뷰/가격/가성비 확인 → **네이버 톡톡**으로 문의

### 1-2. 기타 유입 채널
- **문자**: 나이 많은 고객, 네이버 사용이 어려운 고객 → 전화 상담 후 필요 정보만 문자로 수신
- **카카오톡**: 극소수 (100명 중 10명 미만)
- **전화**: 고령 고객이 직접 전화 → 링크 보내면 이탈하므로 전화로 상담 완료 후 문자로 정보 수집

### 1-3. 네이버 톡톡 초기 자동 응답
- 고객이 톡톡 진입 시 자동으로 초기 문구 노출
- 1번, 2번, 3번 선택지 제공 (시공 문의, 날짜, 관심 제품 등)
- 고객이 선택 후 상담 시작

## 2. 상담 프로세스

### 2-1. 제품 상담
- **Case A: 고객이 제품을 이미 정한 경우** → 해당 제품 견적 바로 제공, 상담 짧음
- **Case B: 제품을 모르는 경우** → 니즈 파악 질문 전송
  - "가성비 / 디자인·기능 / 프리미엄" 중 선택 유도 (복붙 멘트)
  - 선택에 따라 해당 블로그 링크 전송 (제품별 상세 페이지: 사진, 가격대, 장단점 정리)
    - 가성비 라인: 스탠다드 패키지, 리스 패키지, 네오녹스 패키지 등
    - 디자인·기능 라인: 아티잔 시리즈 등
  - 고객이 블로그 글을 읽고 제품 선택 → 견적 진행

### 2-2. 현장 확인 상담
- 고객에게 **현장 치수 및 사진** 요청
- 사진 기반으로 설치 가능 여부 판단
- 단순 제품 판매가 아닌 **설치 현장 변수**가 있으므로 100% AI 상담 전환은 어려운 영역
  - 상판 재질 확인 (인조대리석 / 엔지니어드스톤 / 세라믹 등)
  - 모서리 따내기, 절단 작업 필요 여부
  - 배수구멍 위치, 경첩 간섭 등 현장별 변수

### 2-3. 견적서 발송
- 상담 완료 후 견적서 전송 (예: 509,000원)
- **견적서에 포함되는 할인/프로모션 안내:**
  - 지인 추천 시 → 3만원 페이백
  - 포토리뷰 이벤트 → 5천원 페이백
  - 아파트 단톡방/지역 카페 소개 시 → 소개 1건당 3만원 페이백
- 고객이 견적서 확인 후 "설치하겠다" 의사 표시

## 3. 시공 전 확인 프로세스

### 3-1. 추가 비용 사전 고지
- **"시공 전 꼭 확인해주세요"** 안내문 발송 (클레임 방지 목적으로 지속 업데이트된 문서)
- 추가 비용 발생 가능 항목:
  - **인조대리석 모서리 따내기**: 무상 (플라스틱 재질이라 쉬움)
  - **엔지니어드스톤 모서리 따내기**: 1만원 (석재, 분진 발생 가능)
  - **인조대리석 절단 작업** (길게 따내기): 3만원
  - **엔지니어드스톤 절단 작업**: 5만원
  - **경첩 위치 이동 작업**: 추가 비용 발생
  - **엔지니어드스톤 분진 발생 고지**: 클레임 경험으로 반드시 사전 안내
- 고객이 **"확인 완료"** 응답해야 다음 상담 단계 진행
  - 목적: 나중에 추가 비용 분쟁 시 "사전 고지 및 확인 완료" 근거 확보

### 3-2. 기본 시공비 구조
- **기본 시공비**: 15만원 (고객 부담)
- 사진으로 추가 작업이 예상되면 미리 18만원 등으로 조정
- 현장 상황에 따라 추가 비용 발생 가능 안내

## 4. 예약 확정 및 정보 수집

### 4-1. 고객 정보 수집
- 견적 승인 → 고객에게 정보 남기도록 요청:
  - **성함, 연락처, 주소, 희망 설치 날짜**
- 설치 날짜 조율:
  - 고객 희망일 vs 기사 일정 조율
  - 수도권: 기사 3명 → 유연한 조율 가능
  - 지방(예: 대전): 기사 1명 → 기사 일정에 맞춰야 함
  - 이사 예정 고객: 오늘 상담 → 3월, 4월 설치 등 장기 예약 존재

### 4-2. 예약금 결제
- **네이버 결제 링크** 전송 → 5만원 예약금 결제
- 결제 금액을 크게 잡지 않는 이유: **네이버 수수료 6.6%** 부담
- 예약금 결제 완료 시 캔슬률 극히 낮음 (100명 중 1명 정도)
- **카드 결제 요청 시 대응 멘트** (AI로 작성):
  - "카드 결제 가능하나, 할인 프로모션이 제외되어 3만원 추가"
  - "카드 수수료" 표현 사용 불가 (신고 위험) → "프로모션 제외" 표현 사용
  - "현금영수증 100% 발급, 소득공제로 더 이득" 안내

## 5. AI 발주 양식 생성 (Gemini 활용)

### 5-1. 작동 방식
- 상담 채팅 내용을 **드래그 → Gemini(재미나이)에 붙여넣기**
- "발주" 키워드 입력 시 AI가 **3가지 양식을 자동 생성**:

### 5-2. 양식 ① — 고객 전달용 (예약 정보 확인)
- 고객 정보 요약: 주소, 연락처, 예약 일정
- 결제 링크 포함
- 유의사항 안내
- 고객에게 "예약 정보 확인 후 5만원 결제해주세요" 멘트와 함께 전송

### 5-3. 양식 ② — 내부 발주방용 (정보 공유)
- 고객 정보 + 제품 정보 + 마진 정보 + 특이사항
- 날짜별로 발주 단톡방에 업로드
- 목적: 상담원 간 정보 공유, 상담 중복 방지
  - "내가 상담한 건 내가 올리고, 네가 상담한 건 네가 올리고"
  - 예약 완료된 건만 올림

### 5-4. 양식 ③ — 기사 전달용
- 고객 주소, 연락처, 제품 사이즈, 설치비
- **제품 금액은 기사에게 비공개** (설치비만 노출)
- 카카오 오픈채팅방을 통해 기사에게 전달

## 6. 발주 및 배송

### 6-1. 도매업체 발주
- 도매업체(예: 기침이후로, 티자인 등)에 발주
- 고객 주소 + 제품 정보 복사 → 발주방에 전달
- 업체에서 송장번호 + 명세서(총 금액) 제공 → 송금 처리

### 6-2. 배송지 결정
- **고객집 배송**: 제품 수령 가능한 경우
- **기사집 배송**: 공실, 직장인, 인테리어 중 등 수령 곤란 시 → 기사가 제품 지참하여 방문 설치
- **셀프**: 고객이 직접 설치하거나 인테리어 업체가 시공 → 제품만 판매

### 6-3. 배송 안내 멘트
- "제품은 설치일 2~3일 전 발송"
- "배송 중 파손 가능성 있으니 수령 시 확인 요청"
- "수령/보관이 어려우시면 설치팀장님이 제품 직접 지참 방문 가능"
- 고객 응답에 따라 엑셀에 배송지 구분 기록 (고객집 / 기사집 / 셀프)

### 6-4. 발주 타이밍
- **시공 예정일 기준 3~4일 전** 발주
- 상담은 일찍 했더라도 설치일이 먼 경우(예: 3월, 4월) → 해당 시점에 맞춰 발주

## 7. 엑셀 관리 (현재 수기 작업)

### 7-1. 기록 항목
| 항목 | 내용 |
|------|------|
| 고객명 | 성함 |
| 주소 | 설치 주소 |
| 연락처 | 전화번호 |
| 유입 채널 | 네이버톡 / 문자 / 개인카톡 (나중에 고객 검색용) |
| 제품 정보 | 싱크볼 모델, 수전 모델, 추가 구성품 |
| 매출(판매가) | 고객 결제 금액 |
| 매입단가 | 도매업체 매입가 |
| 마진 | 매출 - 매입 - 수수료 - 페이백 등 |
| 시공 예정일 | 설치 날짜 |
| 발주일 | 시공일 3~4일 전 |
| 설치 담당 기사 | 기사명 + 업체명 |
| 배송지 구분 | 고객집 / 기사집 / 셀프 |
| 시공비 | 기본 15만원 + 추가 작업비 |
| 입금 확인 (고객) | 노란색=미입금, 핑크색=입금 완료 |
| 입금 확인 (기사 페이백) | 회색=미입금, 핑크색=입금 완료 |
| 기사 정산 방향 | 마이너스(-)=우리가 받을 돈, 플러스(+)=우리가 줄 돈 |
| 리뷰 이벤트 | 5천원 페이백 차감 |
| 네이버 수수료 | 5만원 기준 6.6% |
| 카드 수수료 | 전액 카드 결제 시 발생 |

### 7-2. 색깔 코드 시스템
- **노란색**: 시공 완료 but 고객 입금 미확인
- **핑크색**: 입금 확인 완료
- **회색**: 기사 페이백 미입금

### 7-3. 월간 정산
- 엑셀 합산으로 월 매출, 월 매입단가, 월 순수익 산출
- 순수익 기준으로 직원 인센티브 지급
- **현재 문제점**: 수기 입력 → 매입단가 오입력 시 마진 계산 전체 오류, 직원 인센티브 왜곡 가능

## 8. 기사 관리

### 8-1. 기사 등급 및 시공비 구조
- 기사마다 시공비 체계가 다름 (그레이드/협상 결과):

| 기사 | 고객 수금액 | 우리에게 페이백 | 비고 |
|------|-------------|-----------------|------|
| 대전 기사 | 15만원 | 1만원 페이백 | |
| 광주 기사 | 20만원 (절단 5만원 포함) | 2만원 페이백 | |
| 수도권 기사 A | 13만원 | 2만원 페이백 | 단가 낮게 수용 |
| 수도권 기사 B | 15만원 | 2만원 페이백 | |

- 시공비는 **현장에서 고객이 기사에게 직접 계좌이체**
- 기사가 우리에게 페이백 금액을 별도 송금
- 할인 이벤트 시(예: 고객에게 12만원 할인) → 기사 시공비 15만원이면 차액 3만원을 우리가 기사에게 지급

### 8-2. 기사 입금 관리
- 기사별 입금자명이 다를 수 있음 (본인명, 배우자명, 업체명)
- 예: 김용택 기사 → 김유한(아내)으로 입금, 정규욱 기사 → 강유경으로 입금
- 입금자명 매핑 테이블을 엑셀에 별도 관리

### 8-3. 기사 배정 및 일정 관리
- **카카오 오픈채팅**으로 기사에게 일정 전달
- 일정 전달 **2회**:
  1. 예약 확정 시: "2월 28일 설치 있습니다" + 고객 정보
  2. 설치 전날: "내일 일정입니다" + 고객 정보 재전달 (리마인드)
- 전날 리마인드 이유: 기사가 까먹고 방문 안 한 사례 있었음 → 고객 클레임

### 8-4. 기사 운영 원칙 (시행착오 기반)
- **하루 최대 2건** 배정 (3건 이상 시 급해져서 마무리 부실 → 클레임)
- 1건 약 2시간 시공 + 이동 포함 약 3.5~4시간
- 기사별 색깔 코드로 구글 캘린더 관리 (예: 김영택=세, 김인복=노란색, 정규욱=보라색)
- 해피콜로 기사 평가 → 불량 기사 정리

### 8-5. 기사에게 요구하는 사항
- 설치 전날 고객에게 사전 연락 (해피콜 형태)
  - 멘트 예시까지 제공: "내일 ○○시 ○○아파트 1106호 방문 예정입니다"
- 미이행 기사는 일감 배정 중단으로 제재

## 9. 구글 캘린더 운영

### 9-1. 현재 운영 방식
- 일자별 설치 일정을 구글 캘린더에 기록
- 기사별 색깔 구분
- 하루 평균 설치 건수: 주중 2~4건, 주말 최대 10건

### 9-2. 기사와 캘린더 공유 희망 (미구현)
- 기사들이 자기 일정만 볼 수 있도록 공유하고 싶음
- **현실적 장벽**: 기사들이 IT에 익숙하지 않음 (구글 계정 로그인도 어려움)
- 이상적 방향: 캘린더 공유 시 리마인드 이중 전송 불필요 + 기사 자율 확인

## 10. 설치 완료 후 프로세스

### 10-1. 설치 완료 확인
- 기사가 설치 완료 연락 → 우리가 고객에게 **해피콜**
- "시공 잘 받으셨나요? 불편한 점 없으셨나요?"

### 10-2. 리뷰 및 추천 유도
- 포토리뷰 요청 + 링크 전송 → 5천원 페이백
- 지인 소개 / 아파트 단톡방 / 지역 카페 공유 요청 → 1건당 3만원 페이백
- 단톡방 공유 인증 스크린샷 수신

### 10-3. 제품 유의사항 안내
- 해피콜 시 제품 사용 유의사항 함께 전달

### 10-4. 완료 기준
- 해피콜 + 리뷰 이벤트 안내까지 끝나야 **일정 완전 마무리**

## 11. 매출/마진 구조 상세

### 11-1. 제품 가격 예시 (아티잔 네오녹스 858 + 카시아 G7 수전)
| 항목 | 금액 |
|------|------|
| 싱크볼 매입단가 (네오녹스 858) | 253,000원 |
| 수전 매입단가 (카시아 G7) | 48,000원 |
| **총 매입단가** | **301,000원** |
| 고객 판매가 (패키지) | 420,000원 (할인 후 기준) |
| **제품 마진** | **약 119,000원** |

### 11-2. 전체 수익 계산 구조
- 제품 마진 = 판매가 - 매입단가
- 기사 페이백 수입 = 기사별 페이백 금액 (1~2만원/건)
- 차감 항목: 네이버 수수료(6.6%), 카드 수수료, 리뷰 페이백(5천원), 추천 페이백(3만원), 할인 프로모션 차액 부담

### 11-3. 매입단가 관리
- 도매업체와 거래량 증가에 따라 단가 점진적 인하
- 현재 가격 변동 빈도: 거의 없음 (안정적)
- **단가표** 별도 관리 필요

### 11-4. 추가 판매 상품
- 싱크볼 + 수전 외에도: 절수 페달, 욕실 수전, 욕실 샤워기, 코너 선반 등
- 고객이 추가 문의 시 → 도매업체 재고 확인 후 링크 전달
- 대형 설치 시 기사 시공비도 상승 (예: 싱크볼+수전+샤워기2+세면대수전2 = 50만원 시공비, 약 6~7시간)

## 12. 클레임 관리

### 12-1. 주요 클레임 유형
- **분진 클레임**: 엔지니어드스톤 작업 시 분진 날림 → 사전 고지 필수화
- **추가 비용 분쟁**: 현장에서 예상 못한 추가 비용 발생 → 사전 안내문 + 확인 완료 프로세스로 대응
- **대리석 파손**: 시공 중 상판 깨짐 → 기사와 반반 부담 (기사 잘못이지만 파트너 관계 유지)
- **기사 태도**: 급하게 작업 → 마무리 부실, 불친절 → 해피콜로 확인 후 기사 정리

### 12-2. 클레임 대응 원칙
- 고객은 우리를 믿고 한 것 → 기사 잘못이어도 우리가 1차 처리
- 시행착오를 통해 안내문/멘트를 지속 업데이트
- "분명히 알려줬고, 고지했고, 확인 완료까지 받았다" → 법적 근거 확보

## 13. AI 활용 현황

### 13-1. Gemini (재미나이) 활용
- 상담 채팅 내용 → 드래그 → "발주" 입력 → 3가지 양식 자동 생성
- 사전에 학습(프롬프트 세팅)시켜 놓은 결과값 양식대로 출력
- 복붙만 하면 되므로 업무 효율 대폭 향상

### 13-2. AI 멘트 생성
- 카드 수수료 대응 멘트, 고객 안내 멘트 등을 AI에게 상황 설명 후 생성
- 생성된 멘트에서 수정/편집하여 최종 활용

## 14. 자동화 우선순위 (대화 중 합의 사항)

### 최우선 과제 (3가지)
1. **고객 정보 취합 자동화**: 상담 내용 → 엑셀 자동 입력
2. **마진 검증 자동화**: 매입단가 자동 매칭 → 마진 계산 오류 방지, 인센티브 정확도 확보
3. **입금 확인 자동화**: 고객 입금 + 기사 페이백 입금 자동 대조

### 후순위 과제
4. **구글 캘린더 연동**: 예약 정보 → 캘린더 자동 등록 + 기사별 일정 공유
5. **기사 정보 전달 자동화**: 카카오톡으로 기사에게 일정 자동 전송

## 기타 (프로세스 외 대화 내용)

### 1. Claude 관련 대화
- Claude Code / Claude Cowork에 대한 관심 표현
- 월 200달러 결제 고민 중 → 상대방에게 반반(100불씩) 계정 공유 제안
- 현재 Gemini 중심 활용 → Claude Code로 자동화 개발 전환 검토 중

### 2. 기사 관련 에피소드
- 매재(이종우)가 현재 기사로 활동 중 (빨간색 캘린더)
- 한강이라는 지인에게 기사 일을 시키려 했으나, 이후 별도 연락 없음
- 처음에 매재가 급하게 시공하여 클레임 발생 → 하루 2건 제한 정책 수립 계기
- 피쇼(기사)는 어머님과 밥 먹고 온다는 에피소드

### 3. 기사 수입 관련
- 하루 2건 × 평균 15만원 = 하루 30만원 수입
- 실내 작업이라 날씨 영향 없음
- 우리가 일감을 계속 던져주는 구조 → 기사들도 좋은 관계 유지 동기

### 4. 직원(성태) 관련
- 현재 상담 직원 1명 (성태) + 대표(본인)가 함께 운영
- 직원 업무 감독도 부담 → 자동화 시 감독 부담 감소 기대
- 인센티브: 상담 건당 순수익 기반 지급

### 5. 개인 대화
- 상대방이 "설날에 알바하고 있다", "롤(게임) 하려면 음료수 값은 벌어야 한다" 등 잡담
- 펜션 여행 때 만났던 지인(한강) 이야기
