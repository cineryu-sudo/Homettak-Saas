# OneDrive Excel 연동 가능 폼 도구 비교 및 추천

---

## 핵심 결론: 🏆 Microsoft Forms가 가장 적합

OneDrive Excel과 **네이티브(내장) 연동**되는 폼 도구는 사실상 **Microsoft Forms 하나**뿐입니다.
나머지 도구들은 모두 Zapier, Make, Power Automate 같은 중간 자동화 도구가 필요합니다.

## 2. 🏆 1순위 추천: Microsoft Forms

### 왜 Microsoft Forms인가?

**OneDrive Excel과 유일하게 네이티브 연동되는 폼 도구**이기 때문입니다.

- 폼 생성 시 "응답" 탭 → "Excel에서 열기" 클릭만으로 OneDrive에 Excel 파일 자동 생성
- 응답이 들어올 때마다 해당 Excel에 자동으로 행 추가
- 별도의 자동화 도구(Make, Zapier) 필요 없음
- 무료 Microsoft 계정으로 사용 가능 (Microsoft 365 구독 불필요)
- 2024년 10~11월 신규 동기화 솔루션 업데이트로 안정성 향상

### Microsoft Forms + OneDrive Excel 연동 방식

```
방법 A: Forms에서 직접 생성
━━━━━━━━━━━━━━━━━━━━━━━━━
1. forms.office.com 접속 → 폼 생성
2. 응답 탭 → "Excel에서 열기" 클릭
3. OneDrive에 Excel 파일 자동 생성 + 응답 자동 동기화
4. 해당 Excel 파일에 VLOOKUP, 수식 등 자유롭게 추가 가능

방법 B: OneDrive에서 폼 생성 (더 추천)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. OneDrive 접속 → "새로 만들기(+New)" → "Forms 설문조사"
2. OneDrive 내에 Excel 파일 자동 생성
3. 폼 응답이 해당 Excel에 실시간 동기화
4. 기존 OneDrive 폴더 구조에 자연스럽게 통합
```

### Microsoft Forms의 장점 (홈앤뚝딱 기준)

1. **비용 $0**: 무료 Microsoft 계정으로 충분
2. **OneDrive Excel 네이티브 연동**: 중간 도구 없이 폼 응답 → Excel 자동 기록
3. **한국어 완벽 지원**: 고객이 한국어로 불편 없이 사용
4. **Microsoft 생태계 통합**: 현재 쓰는 OneDrive Excel과 완벽 호환
5. **Power Automate 무료 연동**: 향후 자동화 확장 시 가장 매끄러운 연결
6. **이미지 업로드 지원**: 고객 현장 사진 수집 가능
7. **파일 첨부 → OneDrive 자동 저장**: 업로드된 사진이 OneDrive에 자동 저장

### Microsoft Forms의 단점 (솔직하게)

1. **디자인 자유도 낮음**: Tally나 Typeform 대비 폼 디자인이 투박함
   - 하지만 고객이 예약 정보를 입력하는 용도이므로 디자인보다 기능이 중요
2. **조건부 로직 제한**: "분기" 기능은 있지만 복잡한 조건 분기는 어려움
   - 홈앤뚝딱 폼은 복잡한 분기가 필요 없으므로 충분
3. **URL 파라미터 프리필 제한**: 주문ID를 URL에 넣어 자동 입력하는 기능이 제한적
   - 대안: 고객에게 주문ID를 별도로 입력하게 하거나, 폼 제출 시각으로 매칭
4. **실시간 동기화 조건**: Excel 웹 버전에서 열어야 최신 응답 반영
   - OneDrive 데스크톱 동기화 사용 시 자동 반영됨

## 3. 2순위 추천: Tally + Power Automate(무료)

Tally의 디자인 자유도가 꼭 필요하고, Power Automate를 쓸 의향이 있다면 이 조합도 가능합니다.

### 연동 방식
```
Tally 폼 제출
    │
    ├─→ Tally Webhook 발동 (무료)
    │
    ├─→ Power Automate 트리거 수신 (무료 플랜 가능)
    │
    ├─→ "Excel Online(Business)" 커넥터로 OneDrive Excel에 행 추가
    │
    └─→ 완료
```

### 장단점
- 장점: Tally의 예쁜 디자인, URL 파라미터 프리필, 무료
- 단점: Power Automate 설정이 필요 (한 번만 하면 됨), 중간 도구 의존

### 주의사항
- Power Automate 무료 플랜: 월 750회 실행 제한 (월 200건 주문이면 충분)
- "Excel Online(Business)" 커넥터는 OneDrive for Business 필요
- 개인 OneDrive의 경우 "Excel Online(OneDrive)" 커넥터 사용

## 4. 3순위: Jotform (OneDrive 연동은 PDF 저장만)

### 연동 방식
- Jotform → OneDrive 네이티브 연동: **PDF로만 저장** (Excel 행 추가 아님)
- Jotform → OneDrive Excel 행 추가: **Zapier 필요** (유료 $19.99/월~)

### 적합한 경우
- 결제 기능이 폼 내에 필요한 경우 (Jotform은 결제 연동 강력)
- 복잡한 조건부 로직이 필요한 경우

### 비추천 이유
- OneDrive Excel 직접 연동 불가 → Zapier 비용 추가 발생
- 무료 플랜 월 100건 제출 제한 (홈앤뚝딱 월 200건에 부족)

## 5. 홈앤뚝딱 맞춤 추천 구성

### 최종 추천: Microsoft Forms × 2개 + OneDrive Excel

```
┌───────────────────────────────────────────────────────┐
│                                                       │
│  📋 고객용 Microsoft Forms          비용: $0           │
│  (고객이 직접 입력)                                     │
│  → OneDrive Excel [고객정보 시트]에 자동 기록            │
│                                                       │
│  📊 업체용 Microsoft Forms          비용: $0           │
│  (상담원이 입력)                                       │
│  → OneDrive Excel [업체관리 시트]에 자동 기록            │
│                                                       │
│  📁 OneDrive Excel                                    │
│  ├── Sheet: 고객정보 (Forms 자동 기록)                  │
│  ├── Sheet: 업체관리 (Forms 자동 기록)                  │
│  ├── Sheet: 단가표 (수동 관리)                          │
│  ├── Sheet: 기사DB (수동 관리)                          │
│  └── Sheet: 통합 대시보드 (VLOOKUP + 자동 계산)         │
│                                                       │
│  💰 총 비용: $0                                        │
└───────────────────────────────────────────────────────┘
```

### 주문ID 매칭 방법 (Microsoft Forms 기준)

Microsoft Forms는 URL 파라미터 프리필이 제한적이므로, 아래 방법을 권장합니다:

```
방법 1: 상담원이 주문ID 생성 → 고객에게 전달 → 고객이 폼에 직접 입력
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
장점: 가장 단순, 추가 도구 불필요
단점: 고객이 ID를 잘못 입력할 수 있음
대응: 폼 첫 번째 필드에 "상담원에게 안내받은 예약번호를 입력해주세요" 배치
      + Excel에서 COUNTIF로 중복/오류 자동 감지

방법 2: 접수 시각 + 고객명으로 매칭 (주문ID 없이)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
장점: 고객이 별도 ID 입력 불필요
단점: 동명이인 시 혼동 가능 (확률 낮음)
대응: 고객명 + 연락처 뒷자리로 매칭

방법 3 (고급): Power Automate로 자동 주문ID 생성
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
고객 폼 제출 → Power Automate가 주문ID 자동 생성 → Excel에 기록
+ 상담원에게 "HD-250219-003 접수됨" 알림 자동 발송
+ 상담원이 업체 폼에서 해당 ID 선택 (드롭다운)
```

### 업체 폼에서 "드롭다운 자동 계산" 구현 방법

Microsoft Forms 자체에는 드롭다운 선택 시 매입단가 자동 표시 기능이 없습니다.
하지만 **Excel 수식으로 해결** 가능합니다:

```
[업체용 Forms에서 제품명을 "선택형 질문"으로 선택]
    │
    ▼
[OneDrive Excel에 "아티잔 네오녹스 858"로 자동 기록]
    │
    ▼
[Excel 수식: =VLOOKUP(제품명, 단가표시트, 매입단가열, FALSE)]
    → 매입단가 253,000원 자동 표시
    │
[Excel 수식: =판매가 - 매입단가]
    → 마진 자동 계산
```

→ 상담원이 폼에서 입력하면, **Excel에서 자동 계산이 완성되는 구조**
