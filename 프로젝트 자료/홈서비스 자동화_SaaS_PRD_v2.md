## 1. 개요

### 1.1 문제 정의

홈서비스 시공 중개업체(싱크볼, 도배, 바닥, 타일, 에어컨, 배관 등)는 대부분 **1~3인 소규모**로 운영되며, 다음 문제를 겪고 있다:

| 문제 | 현상 | 영향 |
|------|------|------|
| 수동 고객정보 입력 | 네이버톡/전화 상담 후 엑셀에 수기 입력 (건당 5분) | 일 200분+ 낭비 |
| 마진 계산 오류 | 매입가, 할인, 수수료를 수동 계산 → 누락 빈번 | 월 10~15건 마진 오류 |
| 입금 확인 지연 | SMS 확인 → 엑셀 수기 대조 (건당 2분) | 미입금 추적 누락 월 2~3건 |
| 기사 일정 혼선 | 카카오톡 수동 전달 (건당 3분) | 월 3~5건 일정 혼선 |
| 시공 리마인드 부재 | 전날 수동 개별 연락 (일 15분) | 기사 노쇼/지연 리스크 |

### 1.2 솔루션

**주문 접수부터 정산까지 6단계 업무를 자동화하는 올인원 시공 관리 SaaS**

```
[접수] → [상담/견적] → [예약확정] → [발주] → [시공] → [정산]
 AI파싱    마진자동계산   알림톡자동    발주리마인드  기사리마인드  입금매칭
```

### 1.3 제품 비전

> "홈서비스 시공 중개업체의 **엑셀 + 카톡 업무**를 하나의 도구로 대체하여,
> 1인 운영자가 **월 43시간을 절감**하고 본업(영업/마케팅)에 집중하게 한다."

### 1.4 전략 (확정)

| 단계 | 기간 | 형태 | 목적 |
|------|------|------|------|
| **Phase A** | **2주** | Google Sheets + Apps Script | 최소 검증 (업무 프로세스 확인) |
| **Phase B** | **10주** | **Supabase + Next.js 직접 개발** | SaaS 웹앱 MVP 출시 |
| **총 기간** | **3개월 이내** | | 베타 출시 완료 |

### 1.5 확정된 기술 결정

| 결정 사항 | 확정 값 | 비고 |
|----------|--------|------|
| 기술 스택 | **Supabase + Next.js** | 백엔드 코드 최소화 |
| 인증 | **카카오 로그인** | 타겟 고객이 가장 익숙한 방식 |
| 멀티테넌시 | **Supabase 프로젝트 분리** | 업체별 완전 격리 |
| 모바일 | **PWA (반응형 웹)** | 별도 앱 설치 불필요 |
| 배포 | **Vercel** | GitHub push 시 자동 배포 |
| 결제 | **베타 무료 → 정식출시 시 도입** | 토스페이먼츠 정기결제 |
| 개발 체제 | **1인 개발 (Claude Code 활용)** | 외주/협업 없음 |
| MVP 범위 | **F1~F6 완성 MVP** | 접수~대시보드 전체 |
| 정책 관리 | **GitHub (YAML 파일)** | 버전 관리 + 변경 이력 |

## 2. 타겟 사용자

### 2.1 이상적 고객 프로필 (ICP)

| 항목 | 설명 |
|------|------|
| **업종** | 홈서비스 시공 중개/운영 업체 |
| **세부 업종** | 싱크볼/수전, 도배, 바닥재, 타일, 에어컨, 배관, 조명, 방충망 등 |
| **규모** | 1~5인 운영 (대표 1인 + 협력기사 다수) |
| **월 주문량** | 50~500건 |
| **현재 도구** | 엑셀/스프레드시트 + 카카오톡 + 수동 전달 |
| **유입 채널** | 네이버 스마트스토어, 전화, 숨고, 오늘의집 등 |

### 2.2 사용자 페르소나

#### 페르소나 1: 김대표 (1인 싱크볼 시공 중개업체)

- **상황**: 월 200건 주문, 혼자 접수~정산 전담
- **현재 도구**: OneDrive Excel + 카카오톡
- **핵심 고통**: "하루 2시간을 엑셀 정리에 쓰고 있어요. 마진 계산도 자주 틀려요."
- **원하는 것**: "주문 넣으면 알아서 정리되고, 기사한테도 자동으로 알림 갔으면 좋겠어요."

#### 페르소나 2: 박대표 (도배/바닥 시공 업체, 2인 운영)

- **상황**: 월 300건 주문, 직원 1명과 역할 분담
- **현재 도구**: Google Sheets + 수동 관리
- **핵심 고통**: "기사 10명 관리가 복잡해요. 누구한테 뭘 보냈는지 헷갈려요."
- **원하는 것**: "기사 배정하면 자동으로 알림 가고, 일정도 한눈에 보고 싶어요."

### 2.3 비사용자 (Anti-Persona)

- 대형 인테리어 회사 (ERP/자체 시스템 보유)
- 직접 시공만 하는 기사 (중개/관리 업무 없음)
- IT 내재화된 프랜차이즈

### 3.2 기능 상세

#### F1. 주문 접수 자동화

**F1.1 AI 대화 파싱**

| 항목 | 설명 |
|------|------|
| **목적** | 네이버톡/카카오톡 상담 대화를 AI로 자동 파싱하여 주문 정보 입력 |
| **입력** | 대화 내용 전체 복사 → 텍스트 영역에 붙여넣기 |
| **처리** | Gemini API가 고객명, 연락처, 주소, 재질, 희망제품, 시공희망일, 특이사항, 할인요청을 JSON으로 추출 |
| **출력** | 파싱 결과 미리보기 → 수정 가능 → "저장" 클릭 시 DB에 추가 |
| **자동 설정** | 주문ID, 접수일, 유입채널, 진행상태("접수") |

**AI 프롬프트:**
```
"[업종명] 서비스 상담 대화에서
고객명, 연락처, 주소, [업종별 필수필드],
희망제품/서비스, 시공희망일, 특이사항, 할인요청을
JSON으로 추출하세요.
언급되지 않은 항목은 '미확인'으로 표시."
```

**F1.2 전화주문 폼**

| 항목 | 설명 |
|------|------|
| **목적** | 전화 상담 중 모바일에서 빠르게 주문 입력 |
| **필드** | 고객명(필수), 연락처(필수), 설치주소(필수), 재질/유형(드롭다운), 희망시기(드롭다운), 특이사항(선택) |
| **자동** | 제출 시 주문ID 자동생성, 유입채널 "전화" 자동 |

**F1.3 주문ID 규칙**

- 형식: `{업체코드}-YYMMDD-NNN`
- 예시: `HD-260301-001`

#### F2. 상담 & 견적 자동화

**F2.1 제품/매입단가 자동 조회**

- 제품 선택 시 매입단가 자동 표시 (Supabase DB 참조)
- 제품단가표: 제품코드, 제품명, 카테고리, 매입단가, 기본판매가

**F2.2 마진 실시간 계산**

```
순수익 = 제품마진 + 기사페이백 - 할인금액 - 채널수수료
마진율 = 순수익 / 총매출
```

**F2.3 마진 경고 시스템**

| 마진율 | 표시 | 액션 |
|--------|------|------|
| 15%+ | 정상 (초록) | - |
| 10~15% | 주의 (노랑) | 경고 배지 |
| 10% 미만 | 위험 (빨강) | 강한 경고 + 확인 모달 |

**F2.4 채널별 수수료**

| 채널 | 수수료 |
|------|--------|
| 네이버 스마트스토어 | 5.5% (설정 가능) |
| 전화 직접 | 0% |
| 카드결제 | 3.3% (설정 가능) |
| 기타 채널 | 업체별 설정 |

**F3.1 기사 배정**

| **배정 방식** | 드롭다운 선택 |

- 추가작업단가표: 업종별 추가 작업 항목 (모서리따내기, 절단, 타공 등)
- 선택 시 추가작업비 자동 합산

**F3.3 내장 캘린더 뷰**

| 항목 | 설명 |
|------|------|
| **트리거** | 진행상태 → "예약확정" 변경 시 자동 등록 |
| **표시** | 기사별 색상 구분, 제목: "[기사명] 고객명 - 제품명" |
| **기능** | 일/주/월 뷰, 기사별 필터, 드래그 일정 변경 |

**F3.4 기사 배정 알림톡**

| 항목 | 설명 |
|------|------|
| **트리거** | 진행상태 → "예약확정" 변경 시 (기사 배정 완료 후) |
| **채널** | 카카오 알림톡 (솔라피/알리고 경유) |
| **내용** | 일시, 고객명, 연락처, 주소, 제품, 추가작업, 배송방식, 특이사항 |
| **Fallback** | 알림톡 실패 시 SMS 대체 발송 |

**알림톡 템플릿:**
```

고객: 홍길동
연락처: 010-1234-5678
주소: 서울 관악구 ○○아파트 1106호
제품: 아티잔 네오녹스 858 + 카시아 G7
추가작업: 모서리따내기
배송: 고객집 직배송
특이사항: 경첩 이동 필요

문의: ☎ 000-0000-0000
```

**F3.5 시공 전날 리마인드**

| 항목 | 설명 |
|------|------|
| **트리거** | 매일 오전 9시 (Supabase Edge Function cron) |
| **대상** | 시공예정일 = 내일인 건을 기사별 그룹핑 |
| **기사에게** | 알림톡: 내일 시공 건 목록 |
| **대표에게** | 앱 내 알림 + 이메일: 요약 리포트 (총 건수, 미입금 건, 발주 상태) |

**F4.2 발주 리마인드**

| 항목 | 설명 |
|------|------|
| **트리거** | 매일 오전 9시 |
| **조건** | 발주일 = 오늘 AND 진행상태 = 예약확정 |
| **내용** | 발주 필요 건 목록 (주문ID, 고객명, 제품, 배송지) |
| **채널** | 앱 내 알림 + 이메일 |

#### F5. 정산 자동화

**F5.1 입금 반자동 매칭**

| 항목 | 설명 |
|------|------|
| **입력** | 입금자명 + 금액 |

| **처리** | 버튼 클릭 → 예약금/잔금 입금 확인 자동 반영 |

**F5.2 순수익 계산**

```
순수익 = 제품마진 + 기사페이백 - 할인금액 - 채널수수료
```

**F5.3 기사 정산 추적**

| 필드 | 설명 |
|------|------|
| 기사정산 | 완료/미완료 |
| 정산금액 | 시공비 - 페이백 |
| 정산일 | 송금 완료일 |

### 4.1 Supabase 테이블 구조

```
homeservice_db (Supabase 프로젝트 — 업체별 분리)
│
├── orders           ← 주문 (핵심 테이블)
├── products         ← 제품단가표
├── engineers        ← 기사 DB
├── addons           ← 추가작업단가
├── discounts        ← 할인항목표
├── payments         ← 입금 기록

└── settings         ← 업체별 설정 (수수료율, 기준값 등)
```

### 4.2 orders 테이블 (핵심)

#### A. 기본정보 (접수 시) — 9개

| 컬럼 | 타입 | 입력 | 설명 |
|------|------|------|------|
| id | UUID | 자동 | PK |
| order_id | VARCHAR | 자동 | HD-YYMMDD-NNN |
| created_at | TIMESTAMP | 자동 | 접수일시 |
| channel | ENUM | 드롭다운 | 네이버/전화/숨고/기타 |
| channel_order_no | VARCHAR | 수동 | 채널 주문번호 |
| customer_name | VARCHAR | 수동/AI | 고객명 |
| customer_phone | VARCHAR | 수동/AI | 연락처 |
| address | TEXT | 수동/AI | 설치주소 |
| material_type | ENUM | 드롭다운 | 재질/유형 (업종별) |
| notes | TEXT | 수동/AI | 특이사항 |

#### B. 제품/가격 (상담 후) — 6개

| 컬럼 | 타입 | 입력 | 설명 |
|------|------|------|------|
| product_id | FK → products | 드롭다운 | 제품 |
| sub_product_id | FK → products | 드롭다운 | 부자재 |
| addon_ids | FK[] → addons | 드롭다운 | 추가구성품 |

| cost_price | INTEGER | 자동 | 매입단가 합산 |

| 컬럼 | 타입 | 입력 | 설명 |
|------|------|------|------|
| construction_date | DATE | 수동 | 시공예정일 |
| construction_time | TIME | 드롭다운 | 시공시간 |
| engineer_id | FK → engineers | 드롭다운 | 담당기사 |
| base_construction_fee | INTEGER | 자동 | 기본시공비 (기사DB) |
| addon_work_ids | FK[] → addons | 드롭다운 | 추가작업 |
| addon_work_fee | INTEGER | 자동 | 추가작업비 합산 |
| total_construction_fee | INTEGER | 자동 | 기본 + 추가 |
| engineer_payback | INTEGER | 자동 | 기사페이백 (기사DB) |

| 컬럼 | 타입 | 입력 | 설명 |

| channel_fee | INTEGER | 자동 | 채널별 수수료 |
| net_profit | INTEGER | 자동 | 제품마진 + 페이백 - 할인 - 수수료 |

#### E. 진행관리 — 8개

| review_event | ENUM | 수동 | 완료/미완료/해당없음 |
| calendar_synced | BOOLEAN | 자동 | 캘린더 등록여부 |
| notification_sent | BOOLEAN | 자동 | 기사 알림 발송여부 |

## 9. 경쟁 분석

| 유형 | 예시 | 장점 | 단점 |
|------|------|------|------|
| 범용 프로젝트 관리 | 노션, 먼데이닷컴 | 유연함 | 시공 특화 기능 없음 |
| 범용 CRM | 세일즈포스, 허브스팟 | 강력함 | 비싸고 복잡, 소규모 부적합 |
| 시공관리 ERP | 자체 개발 | 도메인 특화 | 대기업 대상 |
| 엑셀/Sheets | Google Sheets | 무료, 익숙 | 자동화 한계 |
| **본 제품** | 홈서비스 SaaS | **소규모 특화, AI, 알림톡** | 신규 진입 |

**차별화 5가지:**

1. **소규모 시공 중개업체 특화** — 1~5인 업체의 실제 업무 흐름에 맞춤
2. **AI 기반 고객정보 자동 입력** — 네이버톡 대화 붙여넣기만으로 자동 정리
3. **카카오 알림톡 내장** — 기사 배정/리마인드 자동 발송
4. **마진 실시간 경고** — 마진율 위험 시 즉시 알림
5. **월 29,000원** — 소상공인이 부담 없는 가격

## 부록 A: 용어 정의

| 용어 | 정의 |
|------|------|
| 시공 중개업체 | 고객과 시공기사를 연결하고, 제품 판매 + 시공 서비스를 패키지로 제공하는 업체 |

| RLS | Row-Level Security. DB에서 행 단위로 접근 권한을 제어하는 보안 방식 |
