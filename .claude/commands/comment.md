주어진 파일의 코드에 한국어로 상세한 주석을 추가해 주세요.

## 대상 파일
$ARGUMENTS

## 규칙

1. **함수/메서드**: 각 함수 위에 해당 함수의 목적, 매개변수, 반환값을 설명하는 주석을 추가
2. **클래스/컴포넌트**: 클래스나 컴포넌트의 역할과 책임을 설명하는 주석을 추가
3. **복잡한 로직**: 알고리즘, 조건문, 반복문 등 복잡한 로직에는 인라인 주석을 추가
4. **상수/변수**: 의미가 불분명한 변수나 중요한 상수에 주석을 추가
5. **API 호출**: API 엔드포인트, 요청/응답 형식에 대한 주석을 추가
6. **타입/인터페이스**: TypeScript 타입이나 인터페이스의 각 필드에 대한 설명을 추가

## 주석 스타일

- 해당 언어의 표준 주석 스타일을 사용 (예: JSDoc, Python docstring, Java Javadoc 등)
- 이미 적절한 주석이 있는 곳에는 중복 주석을 추가하지 않음
- 너무 명확한 코드 (예: `i++`, `return true`)에는 불필요한 주석을 달지 않음
- 주석은 "무엇을" 하는지보다 "왜" 하는지를 중심으로 작성

## 중요

- 기존 코드의 동작을 절대 변경하지 않음 (주석만 추가)
- 기존 주석이 잘못된 경우에만 수정
- 파일을 읽고, 주석을 추가한 뒤, Edit 도구로 수정
